# QRAM Simulator Architecture

本文档描述 QRAM Simulator 项目的整体架构、设计哲学和核心模块。

---

## 1. 概述

### 1.1 项目目标

QRAM Simulator 是一个用于模拟**量子随机存取存储器（QRAM）**和**稀疏态量子计算**的高性能模拟器。它旨在为量子算法研究者和开发者提供一个能够：

- 高效模拟大规模量子系统的工具（利用稀疏态表示）
- 精确建模 QRAM 电路的行为和噪声影响
- 支持 GPU 加速以获得更高的计算性能
- 提供简洁的 Python API 便于快速原型开发

### 1.2 设计哲学

1. **稀疏态优先**：利用量子态的稀疏性，将存储和计算复杂度从指数级降低到与纠缠程度成正比
2. **模块化设计**：清晰的模块边界，便于扩展和维护
3. **性能与易用性并重**：C++ 核心提供性能，Python 绑定提供易用性
4. **噪声感知**：原生支持去极化和退相干噪声模型

### 1.3 核心功能

- **QRAM 模拟**：支持 qutrit-based 和 qubit-based 两种 QRAM 电路实现
- **稀疏态模拟**：基于稀疏向量的量子态表示和门操作
- **基础量子门**：H、X、Y、Z、S、T、CNOT、旋转门等
- **高级算法**：QFT、Grover 搜索、量子算术、块编码、态制备
- **CUDA 加速**：关键操作的 GPU 并行实现
- **Python 绑定**：通过 Pybind11 提供的完整 Python API

## 2. 核心模块

```
QRAM-Simulator/
├── Common/           # 共享基础设施
├── SparQ/            # 稀疏态模拟器核心
├── QRAM/             # QRAM 电路实现
├── SparQ_Algorithm/  # 高层量子算法
├── PySparQ/          # Python 绑定
├── Experiments/      # 实验代码
└── ThirdParty/       # 第三方依赖（Eigen）
```

### 2.1 Common/ - 共享基础设施

**职责**：提供跨模块的通用工具和数据结构

**核心组件**：

| 文件 | 功能 |
|------|------|
| `basic.h` | 基础数学工具、复数运算、哈希函数 |
| `matrix.h` | 稀疏/稠密矩阵的 Eigen 封装和扩展 |
| `logger.h` | 日志系统，支持分级日志和性能计时 |
| `error_handler.h` | 统一的错误处理和异常定义 |
| `typedefs.h` | 项目通用类型别名（QIndex、Complex 等） |
| `cuda/` | CUDA 工具函数和内存管理 |

**设计要点**：
- 所有模块都依赖 Common，形成底层基础层
- 使用 Eigen 作为底层线性代数库
- 提供统一的类型系统确保跨模块一致性

### 2.2 SparQ/ - 稀疏态模拟器核心

**职责**：实现稀疏态量子模拟的核心功能

```
SparQ/
├── include/
│   ├── sparse_state_simulator.h   # 主入口点
│   ├── basic_gates.h              # 基础量子门定义
│   ├── basic_components.h         # 量子寄存器和操作组件
│   ├── qram.h                     # QRAM 操作算子
│   ├── quantum_arithmetic.h       # 量子算术运算
│   ├── qft.h                      # 量子傅里叶变换
│   ├── condrot.h                  # 条件旋转门
│   └── ...
└── src/
    ├── basic_gates.cpp            # 门操作实现
    ├── basic_components.cpp       # 组件实现
    ├── qram.cpp                   # QRAM 算子实现
    └── ...
```

**核心类**：

- **`QuantumState`**（定义于 `basic_components.h`）
  - 稀疏态表示：仅存储非零振幅的基态
  - 支持任意数量的量子比特
  - 内置归一化检查和状态验证

- **`QuantumGate`** 和 **`SparseGate`**
  - `QuantumGate`：抽象基类，定义门操作的通用接口
  - `SparseGate`：稀疏优化的门实现
  - 支持单比特、多比特和控制门

- **`QuantumOperator`**
  - 算子的统一接口
  - 支持复合算子和条件算子
  - 内置噪声模型支持

### 2.3 QRAM/ - QRAM 电路实现

**职责**：提供 QRAM 电路的完整模拟实现

```
QRAM/
├── include/
│   ├── qram_circuit_qutrit.h      # Qutrit-based QRAM 电路
│   └── qram_circuit_qubit.h       # Qubit-based QRAM 电路
└── src/
    ├── qram_circuit_qutrit.cpp
    └── qram_circuit_qubit.cpp
```

**核心类**：

- **`QRAMCircuit`（qutrit/qubit）**
  - 地址线位数（`address_size`）和数据线位数（`data_size`）配置
  - 内存存储：经典数据到量子地址的映射
  - 时步逻辑（`time_step`）：生成 QRAM 操作序列
  - 噪声模型：支持去极化和退相干噪声
  - 执行模式：`run_normal()`、`run_full()`、`run_good_only()`

- **`QRAMNode` 和 `QRAMState`**
  - 内部数据结构，表示 QRAM 的层次树结构
  - 支持条件交换和旋转操作

- **`CuQRAMCircuit`**（CUDA 版本）
  - 继承自 `QRAMCircuit`
  - GPU 内存镜像：`memory_dev`（`thrust::device_vector`）
  - 与 SparQ CUDA 算子无缝集成

### 2.4 SparQ_Algorithm/ - 高层算法

**职责**：基于 SparQ 和 QRAM 实现高级量子算法

**包含算法**：

| 算法 | 文件 | 描述 |
|------|------|------|
| 态制备 | `state_preparation.h` | 基于 QRAM 的量子态制备 |
| 块编码 | `block_encoding.h` | 矩阵的量子块编码实现 |
| Grover 搜索 | `grover.h` | Grover 搜索算法 |
| 量子漫步 | `quantum_walk.h` | 量子随机漫步实现 |
| 量子模拟 | `hamiltonian_simulation.h` | 哈密顿量模拟 |
| 误差过滤 | `error_filtration.h` | 错误过滤技术 |

**设计模式**：
- 所有算法都继承自 `QuantumAlgorithm` 基类
- 统一的 `run()` 接口
- 支持算法级别的噪声注入

### 2.5 PySparQ/ - Python 绑定

**职责**：提供 Python API，使 Python 用户能够使用所有 C++ 功能

```
PySparQ/
├── pysparq/
│   ├── __init__.py
│   ├── _core.pyi          # 类型提示文件
│   └── _core.so           # 编译后的绑定（构建后）
├── src/
│   └── pybind_wrapper.cpp # Pybind11 绑定代码
└── pyproject.toml
```

**绑定内容**：
- `QuantumState` → Python 类
- `QuantumGate` 及其子类 → Python 类
- `QRAMCircuit_qutrit/qubit` → Python 类
- 所有算子（`QRAMLoad`、`QRAMLoadFast` 等）
- 类型提示文件（`_core.pyi`）提供 IDE 支持

## 3. 数据流

### 3.1 量子态表示

```
┌─────────────────────────────────────────────────────────┐
│                    QuantumState                          │
├─────────────────────────────────────────────────────────┤
│  稀疏存储: map<QIndex, Complex> amplitudes              │
│  - QIndex: 基态的整数索引 (0 到 2^n-1)                  │
│  - Complex: 复数振幅                                    │
│  - 仅存储 |amplitude| > ε 的非零项                      │
└─────────────────────────────────────────────────────────┘
```

**关键特性**：
- 利用量子态的稀疏性：通常只有少数基态具有非零振幅
- 内存使用：O(k × n)，k 为非零项数，n 为量子比特数
- 对比稠密表示的 O(2^n)，可实现更大规模的模拟

### 3.2 门操作执行流程

```
输入: QuantumState state, QuantumGate gate
         │
         ▼
┌──────────────────────────────────────────────────────┐
│ 1. 确定受影响的基态                                   │
│    - 解析门的控制位和目标位                           │
│    - 确定需要更新的基态集合                           │
└──────────────────────────────────────────────────────┘
         │
         ▼
┌──────────────────────────────────────────────────────┐
│ 2. 应用门操作                                         │
│    - 对每个受影响的基态                               │
│    - 计算新的振幅值                                   │
│    - 更新稀疏映射                                     │
└──────────────────────────────────────────────────────┘
         │
         ▼
┌──────────────────────────────────────────────────────┐
│ 3. 后处理                                             │
│    - 移除接近零的项（阈值清理）                       │
│    - 可选：归一化检查                                 │
└──────────────────────────────────────────────────────┘
         │
         ▼
输出: 更新后的 QuantumState
```

### 3.3 QRAM 加载流程

```
初始状态: |a⟩|z⟩  (地址寄存器 |a⟩, 数据寄存器 |z⟩)
              │
              ▼
┌──────────────────────────────────────────────────────┐
│ QRAMLoad 算子                                        │
│                                                      │
│  1. 获取 QRAM 电路引用                               │
│  2. 验证寄存器尺寸与 QRAM 配置匹配                   │
│  3. 执行 QRAM 加载操作                               │
│                                                      │
│  操作: |a⟩|z⟩ → |a⟩|z ⊕ d[a]⟩                      │
│  - d[a]: 存储在地址 a 的经典数据                     │
│  - ⊕: 按位异或（模2加法）                            │
└──────────────────────────────────────────────────────┘
              │
              ▼
最终状态: |a⟩|z ⊕ d[a]⟩
```

**QRAM 电路内部执行**（qutrit-based）：

```
对于每个时间步 t:
    ┌──────────────────────────────────────────────────┐
    │ 1. 生成该时步的量子操作                            │
    │    - 根据地址和内存状态确定门序列                  │
    │    - 包括条件旋转和交换                            │
    └──────────────────────────────────────────────────┘
                     │
                     ▼
    ┌──────────────────────────────────────────────────┐
    │ 2. 应用噪声（如果启用）                            │
    │    - Depolarizing 噪声                            │
    │    - Amplitude Damping 噪声                       │
    └──────────────────────────────────────────────────┘
                     │
                     ▼
    ┌──────────────────────────────────────────────────┐
    │ 3. 执行量子操作                                    │
    │    - 更新 QRAM 内部状态                            │
    │    - 维护分支概率                                  │
    └──────────────────────────────────────────────────┘
```

## 4. 关键设计决策

### 4.1 为什么使用稀疏态模拟

**背景**：
传统量子模拟器使用稠密向量表示量子态，存储 2^n 个复数。当 n > 30 时，这变得不可行。

**稀疏态的优势**：

1. **内存效率**
   - 许多量子算法（如 Grover、QFT）在大部分计算过程中只探索指数级小的一部分基态
   - 稀疏表示内存使用与纠缠程度成正比，而非量子比特数

2. **计算效率**
   - 门操作只需处理非零振幅的基态
   - 对于浅层电路，计算复杂度从 O(2^n) 降低到 O(poly(n))

3. **适用场景**
   - QRAM 操作：通常涉及稀疏的地址-数据映射
   - 量子漫步：在图上的局部探索
   - 哈密顿量模拟：稀疏哈密顿量

**权衡**：
- 当量子态变得高度纠缠时（所有基态都有显著振幅），稀疏表示失去优势
- 当前实现最适合具有结构化稀疏性的算法

### 4.2 Qutrit-based vs Qubit-based QRAM

**Qutrit-based QRAM**（主要实现）：

```
优势:
┌─────────────────────────────────────────────────────────┐
│ • 更自然的树结构表示                                    │
│   - 每个内部节点有 3 个子节点（对应 qutrit 的 3 态）   │
│   - 与 QRAM 的二叉树结构天然匹配                        │
│                                                         │
│ • 更高效的电路深度                                      │
│   - 某些操作需要更少的门                                │
│   - 减少总门数量，降低噪声累积                          │
│                                                         │
│ • 更灵活的寻址方式                                      │
│   - 支持三进制逻辑                                      │
│   - 某些地址计算更高效                                  │
└─────────────────────────────────────────────────────────┘
```

**Qubit-based QRAM**（备选实现）：

```
优势:
┌─────────────────────────────────────────────────────────┐
│ • 硬件兼容性                                            │
│   - 目前大多数量子计算机基于 qubit                      │
│   - 更容易映射到实际硬件                                │
│                                                         │
│ • 更成熟的理论                                          │
│   - 量子计算理论主要基于 qubit                          │
│   - 更多可重用的算法组件                                │
└─────────────────────────────────────────────────────────┘
```

**设计决策**：
- 以 qutrit-based 为主要实现，追求模拟效率
- 同时提供 qubit-based 实现以保证硬件兼容性
- 两种实现共享相同的 API 接口，可互换使用

### 4.3 CUDA 集成方式

**架构**：

```
┌─────────────────────────────────────────────────────────┐
│                    Host (CPU)                           │
│  ┌─────────────────────────────────────────────────┐   │
│  │  SparQ Core                                     │   │
│  │  - QuantumState（稀疏映射存储在 CPU）            │   │
│  │  - 门操作调度                                   │   │
│  └─────────────────────────────────────────────────┘   │
│                         │                               │
│                    数据传输                             │
│                    （选择性）                           │
│                         │                               │
└─────────────────────────┼───────────────────────────────┘
                          │
┌─────────────────────────┼───────────────────────────────┐
│                    Device (GPU)                         │
│                         ▼                               │
│  ┌─────────────────────────────────────────────────┐   │
│  │  CUDA Kernels                                   │   │
│  │  - 并行门应用                                   │   │
│  │  - 并行振幅更新                                 │   │
│  │  - 并行归一化                                   │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

**策略**：

1. **混合执行**
   - 控制流在 CPU 上执行
   - 数据并行操作在 GPU 上执行
   - 最小化 CPU-GPU 数据传输

2. **选择性加速**
   - 仅当计算量足够大时才使用 GPU
   - 小规模状态保持在 CPU 以避免传输开销

3. **内存管理**
   - 使用 `thrust::device_vector` 管理 GPU 内存
   - 实现自定义分配器减少内存碎片

4. **与 QRAM 的集成**
   - `CuQRAMCircuit` 在 GPU 上维护内存镜像
   - QRAM 加载操作可直接在 GPU 上执行
   - 支持 CUDA 流实现异步执行

## 5. 扩展性

### 5.1 添加新量子门

1. 在 `SparQ/include/` 创建新的头文件
2. 继承 `QuantumGate` 或 `SparseGate` 基类
3. 在 `SparQ/src/` 实现 `apply()` 方法
4. 在 `basic_gates.h` 中注册（如果是基础门）

### 5.2 添加新算法

1. 在 `SparQ_Algorithm/include/` 创建算法头文件
2. 继承 `QuantumAlgorithm` 基类
3. 实现 `prepare()`、`run()` 和 `measure()` 方法
4. 在 `Experiments/` 添加测试程序

### 5.3 添加 Python 绑定

1. 在 `PySparQ/src/pybind_wrapper.cpp` 添加绑定代码
2. 在 `_core.pyi` 添加类型提示
3. 重新构建 PySparQ 模块

PySparQ.pysparq.dynamic_operator.compiler
=========================================

.. py:module:: PySparQ.pysparq.dynamic_operator.compiler

.. autoapi-nested-parse::

   运行时 C++ 代码编译系统

   提供动态编译用户自定义 C++ 算子的功能，支持：
   - 自动生成代码框架
   - 调用 g++ 编译为共享库 (.so)
   - 代码哈希缓存机制，避免重复编译
   - 编译错误捕获和格式化

         // Python 调用辅助函数 - 应用算子到 SparseState
         // Python 侧通过 state._cpp_ptr() 获取 C++ SparseState* 指针，
         // ctypes 将其作为 ctypes.c_void_p 传递。
         extern "C" void apply_operator(BaseOperator* op, SparseState* state) {{
             if (op && state) {{
                 (*op)(*state);
             }}
         }}
         
         // Python 调用辅助函数 - 应用 dagger
         extern "C" void apply_operator_dag(BaseOperator* op, SparseState* state) {{
             if (op && state) {{
                 op->dag(*state);
             }}
         }}
         
         // 获取基类类型
         extern "C" const char* get_base_class() {{
             return "{BASE_CLASS}";

.. py:function:: compile_cpp_code(cpp_code: str, class_name: str, cache_dir: Optional[str] = None, ctor_params: str = '', ctor_args: str = '', config: Optional[CompilerConfig] = None, project_root: Optional[str] = None, verbose: bool = False) -> str

   编译 C++ 代码为共享库

   :param cpp_code: 用户提供的 C++ 代码（包含类定义）
   :param class_name: 算子类名
   :param cache_dir: 缓存目录（默认使用系统临时目录）
   :param ctor_params: 构造函数参数声明
   :param ctor_args: 构造函数参数调用
   :param config: 编译器配置
   :param project_root: 项目根目录（自动检测）
   :param verbose: 是否输出详细日志

   :returns: 编译后的共享库路径 (.so 文件)

   :raises CompilationError: 编译失败
   :raises FileNotFoundError: 找不到编译器

.. py:function:: find_project_root() -> Optional[pathlib.Path]

   查找项目根目录或已安装的包目录

   对于已安装的包，目录结构为:
   - site-packages/pysparq/ (Python包)
   - site-packages/include/ (头文件，包含 basic_components.h)

   对于源代码目录:
   - 项目根目录包含 SparQ/ 和 PySparQ/

   :returns: 项目根目录路径或已安装的包目录，未找到返回 None
